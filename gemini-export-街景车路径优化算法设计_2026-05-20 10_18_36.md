## 3. Algorithm Design and Environmental Modeling (English Version)

This section presents the formal algorithmic design of the proposed Language-Empowered Dynamic Edge-centric Graph Network (LE-DEGN) for street-view vehicle routing in mixed unidirectional/bidirectional urban networks. The LE-DEGN system addresses the Mixed Chinese Postman Problem (MCPP) through a modular architecture comprising: (i) a realistic road network environment with physical-kinematic constraints (Section 3.1-3.5), (ii) an anytime performance evaluation framework (Section 3.6), (iii) a line graph transformation with edge-centric feature encoding (Section 3.7), (iv) an Edge-centric Route Foundation Model (ERFM) based on penalty-aware self-attention (Section 3.8), (v) a Spatio-Temporal Attention-based Dynamic Graph Wave Network (SA-DGWN) for congestion prediction (Section 3.9), (vi) a Language-driven Heuristic Hub (LHH) for adaptive escape planning (Section 3.10), (vii) a REINFORCE-based training pipeline (Section 3.11), (viii) a three-phase integrated execution system (Section 3.12), and (ix) a baseline comparison method (Section 3.13). All formulations are strictly aligned with the implemented Python codebase (`le_degn_system.py`).

### 3.1 Mathematical Formulation of the Mixed Urban Road Network

To address the MCPP under heterogeneous traffic constraints, the urban street network is mathematically represented as a multi-directed graph:

$$\mathcal{G} = (\mathcal{V}, \mathcal{E}, \mathcal{W})$$

where $\mathcal{V} = \{v_1, v_2, \dots, v_n\}$ represents the set of street junctions, $\mathcal{E} \subseteq \mathcal{V} \times \mathcal{V}$ denotes the set of multi-directed edges, and $\mathcal{W}$ represents the set of spatial, physical, and semantic attribute vectors mapped to each edge $e \in \mathcal{E}$.

Node coordinates are distributed across a perturbed 2D grid space. For a target node count $|\mathcal{V}|$, the grid dimensions are computed dynamically:

$$N_x = \max\left(3, \left\lceil\sqrt{|\mathcal{V}|}\right\rceil\right), \quad N_y = \max\left(3, \left\lceil \frac{|\mathcal{V}|}{N_x} \right\rceil\right)$$

Each node $v_k \in \mathcal{V}$ corresponding to grid index $(i, j)$ is assigned coordinates:

$$x_k = j \cdot s + \delta_x, \quad y_k = -i \cdot s + \delta_y$$

where $s = 50.0 / \max(N_x, N_y)$ and $\delta_x, \delta_y \sim \mathcal{N}(0, (0.25s)^2)$. This Gaussian perturbation breaks grid symmetry, forcing the solver to learn robust spatial-geometric heuristics rather than exploiting grid-invariant shortcuts.

### 3.2 Topological Junction Sampling and Edge Connection

The degree connectivity of each junction is regulated by its local topological classification via roulette-wheel sampling:

$$Type(v_k) \sim \text{Categorical}(p_{dead}=0.07, p_T=0.65, p_{cross}=0.18)$$

with corresponding degree caps:

$$\text{deg}_{max}(v_k) = \begin{cases} 1, & \text{if } Type(v_k) = \text{'dead'} \\ 3, & \text{if } Type(v_k) = \text{'T'} \\ 4, & \text{if } Type(v_k) = \text{'cross'} \end{cases}$$

A primary spinal corridor is constructed along $j = \lfloor N_x / 2 \rfloor$ with wide lanes (15-25m). Non-spinal segments have lengths $l_e \sim \mathcal{N}(80, 20^2)$ meters and heading angles $\theta_e = (\theta_{base} + \eta) \bmod 360$, where $\theta_{base} \in \{0^\circ, 90^\circ, 180^\circ, 270^\circ\}$ and $\eta \sim \mathcal{N}(0, 15^2)$. Edge widths are sampled from a discrete distribution: $\{2.5, 3.0, 3.5, 4.0, 4.5, 5.0, 6.0, 8.0\}$ meters with probabilities $\{0.20, 0.25, 0.20, 0.15, 0.10, 0.05, 0.03, 0.02\}$.

### 3.3 Semantic Road Typologies and Temporal Constraints

Each edge $e \in \mathcal{E}$ is assigned one of five urban scenarios with unique speed factor $\beta_e$, congestion probability $P_{e, cong}$, and active time window $[E_e, L_e]$:

| Scenario | $\beta_e$ | $P_{e, cong}$ | Time Window | Characteristic |
|:---|:---:|:---:|:---|:---|
| Main Road | 1.2 | 0.2 | [0, 1440] | High speed, low congestion |
| Commercial Pedestrian | 0.5 | 0.4 | [0, 1440] | Low speed, moderate congestion |
| School Zone | 0.6 | 0.5 | [540, 1020] | Peak-hour congestion |
| Residential | 0.7 | 0.3 | [0, 1440] | Moderate speed, low congestion |
| Construction | 0.3 | 0.8 | [0, 1440] | Very low speed, high congestion |

The spatial-dependent sampling distribution is:

$$P(\text{scenario}) = \begin{cases} [0.1, 0.1, 0.5, 0.2, 0.1], & \text{if } |j - \lfloor N_x / 2 \rfloor| \le 1 \\ [0.1, 0.1, 0.1, 0.6, 0.1], & \text{otherwise} \end{cases}$$

The base traversal cost of edge $e$ is $cost_{e, base} = l_e / \beta_e$.

### 3.4 Physical-Kinematic Translation and Turn Penalties

The system enforces three deterministic physical-kinematic rules:

1. **Width-Based Unidirectional Conversion**: For $w_e < 4.5\text{ m}$, the reverse edge is removed: $\mathcal{E} \leftarrow \mathcal{E} \setminus \{(v, u)\}$.

2. **Narrow Lane Friction**: For $3.0 \le w_e < 4.5\text{ m}$: $cost_e = cost_{e, base} \times 1.5$.

3. **Turn Penalties**: Left turns incur $\Delta \tau_{left} = 1.5\text{ s}$, U-turns incur $\Delta \tau_{uturn} = 7.0\text{ s}$.

These rules shape a highly non-linear optimization landscape, with the extreme U-turn penalty acting as a barrier parameter that shifts the solver away from local minima involving excessive backtracking.

### 3.5 Strong Connectivity Patching

The system executes a two-phase connectivity patching routine to guarantee strong connectivity:

**Phase 1: Zero-Degree Node Resolution.** For each isolated node $n$, the geographically nearest active node $m$ is identified:

$$m = \arg\min_{k \in \mathcal{V}\setminus\{n\}, \text{deg}(k) > 0} \text{hypot}(x_n - x_k, y_n - y_k)$$

Bidirectional patch edges $(n, m)$ and $(m, n)$ are added with default parameters (weight=80s, width=5m, 24h window).

**Phase 2: Sink Node Resolution.** For each sink $s$ with out-degree 0, a directed edge $(s, m)$ is added to its nearest neighbor $m \neq s$.

**Phase 3: Strong Component Bridging.** The system iteratively connects disjoint strongly connected components (SCCs) by bridging between the largest SCC $S_1$ and the next $S_2$, selecting the geographically closest cross-component edge pair:

$$(u^*, v^*) = \arg\min_{u \in S_1, v \in S_2} \text{hypot}(x_u - x_v, y_u - y_v)$$

Bidirectional patch edges are added, and the process repeats until $\mathcal{G}$ is verified strongly connected via Tarjan's algorithm.

**Post-Patching.** An all-pairs shortest path matrix $\mathbf{D}$ is precomputed using Dijkstra's algorithm:

$$\mathbf{D}[u, v] = \text{shortest\_path\_cost}(u, v), \quad \forall u, v \in \mathcal{V}$$

### 3.6 Anytime Performance Evaluation and AOCC Metric

The `PerformanceTracker` records the convergence trajectory $\mathcal{T} = \{(t_0, c_0), \dots, (t_m, c_m)\}$ by monitoring the best-found cost $c_{best}$ over elapsed time $t_{elapsed}$. Updates occur at every cost improvement or at fixed intervals (0.05s).

The Area Over the Convergence Curve (AOCC) quantifies convergence quality. The time budget $T_{budget}$ is discretized into $N = 500$ intervals. At each step $t_k$, the cost is clamped and normalized:

$$c_{clamped}(t_k) = \max(L, \min(c(t_k), U))$$
$$y_{norm}(t_k) = \frac{c_{clamped}(t_k) - L}{U - L}$$

The AOCC is the average of inverted normalized costs:

$$\text{AOCC} = \frac{1}{N} \sum_{k=1}^{N} \left(1.0 - y_{norm}(t_k)\right)$$

The lower bound $L$ is computed as the service cost plus the sum of each row's minimum transition cost (a nearest-neighbor lower bound). The upper bound $U$ is conservatively set as the service cost plus 300 times the number of tour edges.

AOCC near 1.0 indicates near-instant convergence; AOCC near 0.0 indicates stagnation. This integral metric avoids the temporal bias of endpoint-only comparisons and directly quantifies "anytime" resilience.

### 3.7 Line Graph Construction and Edge-Centric Representations

The core innovation of LE-DEGN is the transformation of the primal road graph into a line graph (edge dual graph), enabling the routing problem to be reformulated as a node sequencing problem on edges rather than junctions. The `LineGraphBuilder` performs this transformation.

#### 3.7.1 Line Graph Topology

Each directed edge $e = (u, v, k)$ in the primal graph $\mathcal{G}$ becomes a node in the line graph $\mathcal{L} = (\mathcal{V}_L, \mathcal{E}_L)$:

$$\mathcal{V}_L = \{e \in \mathcal{E}_{\mathcal{G}}\}, \quad |\mathcal{V}_L| = N$$

A directed edge in the line graph exists from edge $e_i$ to $e_j$ if the destination node of $e_i$ equals the source node of $e_j$ (i.e., the vehicle can traverse $e_i$ then immediately enter $e_j$):

$$\mathcal{E}_L = \{(e_i, e_j) \mid v_i^{dest} = v_j^{src}\}$$

#### 3.7.2 Edge Feature Encoding

Each line graph node $e_i$ is encoded with a 13-dimensional feature vector:

$$\mathbf{x}_i = \left[\frac{w_i}{200}, \frac{wid_i}{25}, \frac{\theta_i}{360}, type_i, \frac{x_u}{60}, \frac{y_u}{60}, \frac{x_v}{60}, \frac{y_v}{60}, \frac{d_{in}(u)}{10}, \frac{d_{out}(v)}{10}, \frac{tp_i}{7}, \frac{E_i}{1440}, \frac{L_i}{1440}\right]$$

where $w_i$ is the edge weight, $wid_i$ is the road width, $\theta_i$ is the heading angle, $(x_u, y_u)$ and $(x_v, y_v)$ are endpoint coordinates, $d_{in}(u)/d_{out}(v)$ are node degrees, $tp_i$ is the turn penalty, and $[E_i, L_i]$ is the time window.

#### 3.7.3 Transition Cost Matrix

The critical optimization target is encoded in the transition cost matrix $\mathbf{T} \in \mathbb{R}^{N \times N}$:

$$T_{ij} = \begin{cases} 0, & \text{if } i = j \\ \mathbf{D}[v_i^{dest}, v_j^{src}] + \text{pen}_{turn}(i, j), & \text{if } v_i^{dest} = v_j^{src} \\ \mathbf{D}[v_i^{dest}, v_j^{src}], & \text{otherwise} \end{cases}$$

where $\mathbf{D}[\cdot, \cdot]$ queries the precomputed shortest path matrix, and $\text{pen}_{turn}(i, j) = (\text{turn\_penalty} - 1.0) \times 15.0$ amplifies the kinematic penalty when edges are directly adjacent.

#### 3.7.4 Turn Penalty Matrix

A penalty bias matrix $\mathbf{M} \in \mathbb{R}^{N \times N}$ guides the attention mechanism to avoid kinematically unfavorable transitions:

$$M_{ij} = \begin{cases} -8.0, & \text{if turn penalty} \ge 7.0 \text{ (U-turn)} \\ -1.5, & \text{if turn penalty} \ge 1.5 \text{ (left turn)} \\ 0, & \text{otherwise} \end{cases}$$

The adjacency matrix $\mathbf{A} \in \{0, 1\}^{N \times N}$ encodes the line graph connectivity: $A_{ij} = 1$ iff $(e_i, e_j) \in \mathcal{E}_L$.

### 3.8 Edge-centric Route Foundation Model (ERFM)

The ERFM serves as the global route planning backbone, combining penalty-aware self-attention with an edge-centric decoding mechanism. The total cost of a tour $\pi = (e_{\pi_1}, \dots, e_{\pi_K})$ is decomposed into:

$$C_{total}(\pi) = \underbrace{\sum_{e \in \pi} w(e)}_{\text{Service Cost (fixed)}} + \underbrace{\sum_{k=1}^{K-1} T_{\pi_k, \pi_{k+1}} + T_{\pi_K, \pi_1}}_{\text{Transition Cost (optimizable)}}$$

Since the service cost is invariant to permutation, the optimization reduces to minimizing the transition cost.

#### 3.8.1 Penalty-Biased Multi-Head Attention

The attention mechanism incorporates the penalty matrix as an additive bias to suppress unfavorable edge transitions:

$$\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}, \mathbf{M}) = \text{softmax}\left(\frac{\mathbf{Q}\mathbf{K}^T}{\sqrt{d_k}} + \mathbf{M}\right)\mathbf{V}$$

where $\mathbf{Q}, \mathbf{K}, \mathbf{V}$ are linear projections of input embeddings, $d_k$ is the per-head dimension, and $\mathbf{M}$ provides a negative bias (up to -8.0 for U-turns) to the raw attention scores.

#### 3.8.2 Edge Transformer Block

Each transformer layer consists of:

$$\mathbf{h}' = \text{LayerNorm}\left(\mathbf{x} + \text{Dropout}(\text{Attn}(\mathbf{x}, \mathbf{M}))\right)$$
$$\mathbf{h}_{out} = \text{LayerNorm}\left(\mathbf{h}' + \text{Dropout}(\text{FFN}(\mathbf{h}'))\right)$$

where FFN is a two-layer feed-forward network with GELU activation:

$$\text{FFN}(\mathbf{h}) = \mathbf{W}_2 \cdot \text{GELU}(\mathbf{W}_1 \mathbf{h} + \mathbf{b}_1) + \mathbf{b}_2$$

The ERFM encoder comprises 3 stacked Edge Transformer Blocks with model dimension $d_{model} = 64$, $n_{heads} = 4$, and FFN dimension $d_{ff} = 128$.

#### 3.8.3 Global Attribute Embedding

A global attribute vector $\mathbf{g} \in \mathbb{R}^4$ encodes network-wide statistics:

$$\mathbf{g} = \left[\frac{\bar{w}}{25}, \text{left\_penalty}, \text{uturn\_penalty}, 0.5\right]$$

This vector is projected through a two-layer MLP and broadcast-added to all edge embeddings, providing global context to each local edge representation.

#### 3.8.4 Decoder with Value Estimation

The decoder operates autoregressively. Given the current edge embedding $\mathbf{e}_{curr}$, the context embedding $\mathbf{e}_{ctx}$ (mean of visited edge embeddings), and candidate edge embeddings $\mathbf{e}_{cand}$, the probability distribution over next-edge choices is:

$$q = \mathbf{W}_q \cdot [\mathbf{e}_{curr} \oplus \mathbf{e}_{ctx}]$$
$$\mathbf{s} = \frac{q \cdot (\mathbf{W}_k \mathbf{e}_{cand})^T}{\sqrt{d_{model}}}$$
$$\pi_{\theta}(e_{next} \mid \cdot) = \text{softmax}(\mathbf{s} / \tau)$$

where $\tau$ is a temperature parameter controlling exploration, and $\oplus$ denotes concatenation.

A parallel value head estimates the state value for reinforcement learning:

$$V(\mathbf{e}_{curr}, \mathbf{e}_{ctx}) = \text{MLP}([\mathbf{e}_{curr} \oplus \mathbf{e}_{ctx}])$$

### 3.9 Spatio-Temporal Attention-based Dynamic Graph Wave Network (SA-DGWN)

SA-DGWN predicts future congestion probabilities for each road segment, enabling proactive route adaptation. The network processes historical traffic signals (speed and flow) through stacked spatio-temporal blocks.

#### 3.9.1 Dilated Causal Convolution

The temporal component uses 1D dilated causal convolutions to capture temporal dependencies with exponentially growing receptive fields:

$$h_t^{(l)} = \sum_{k=0}^{K-1} \mathbf{W}_k^{(l)} \cdot \mathbf{x}_{t - d^l \cdot k} + \mathbf{b}^{(l)}$$

where $d^l = 2^l$ is the dilation rate at block $l$, extending the receptive field without increasing parameter count.

#### 3.9.2 Graph Convolution Layer

The spatial component employs a normalized graph convolution:

$$\mathbf{h}^{(spatial)} = \text{ReLU}\left(\mathbf{W} \cdot \hat{\mathbf{A}} \cdot \mathbf{x}\right)$$

where $\hat{\mathbf{A}} = \mathbf{A} / \mathbf{D}$ is the row-normalized adjacency matrix, propagating information along the line graph topology.

#### 3.9.3 Spatio-Temporal Block with Gating

Each SpatioTemporalBlock integrates temporal and spatial processing with a gating mechanism:

$$\mathbf{h}_{tcn} = \text{DilatedCausalConv}(\mathbf{x})$$
$$\mathbf{h}_{gcn} = \text{GraphConv}(\mathbf{h}_{tcn}[:, -1, :], \mathbf{A})$$
$$\mathbf{g} = \sigma(\mathbf{W}_g \cdot \mathbf{h}_{gcn})$$
$$\mathbf{h}_{out} = \text{LayerNorm}(\mathbf{h}_{gcn} \odot \mathbf{g})$$

The sigmoid gate $\mathbf{g} \in [0, 1]^d$ adaptively modulates each feature dimension, enabling the network to selectively emphasize informative spatio-temporal patterns.

#### 3.9.4 Training Procedure

The SA-DGWN is trained on synthetic traffic data with 12 historical timesteps of speed and flow features. Training labels are binary congestion indicators derived from speed thresholds (speed < 0.3 → congestion). The loss function is binary cross-entropy:

$$\mathcal{L}_{BCE} = -\frac{1}{N}\sum_{i=1}^{N} \left[y_i \log \hat{y}_i + (1 - y_i) \log(1 - \hat{y}_i)\right]$$

The output layer uses a Sigmoid activation to produce congestion probabilities $\hat{p}_i \in [0, 1]$, where edges with $\hat{p}_i > 0.5$ are classified as high-risk.

### 3.10 Language-driven Heuristic Hub (LHH)

The LHH module provides adaptive escape planning when dynamic congestion blocks the current route. It formalizes a "Generate, Score, Execute, Self-reflect" (GSES) loop over five heuristic templates.

#### 3.10.1 State Verbalization

Before escape, the `StateVerbalizer` constructs a structured textual representation of the local subgraph (k-hop neighborhood) including node positions, edge widths, weights, and blockage status, providing context for heuristic selection.

#### 3.10.2 Heuristic Template Library

Five complementary heuristic templates are maintained:

| # | Heuristic | Strategy | Max Depth | Key Parameter |
|:---:|:---|:---|:---:|:---|
| 1 | Greedy Widest Road | Maximize edge width | 15 | Width priority |
| 2 | Min Penalty Escape | Minimize turn penalty × weight | 15 | Left×1.5, U-turn×7.0 |
| 3 | Max Connectivity | Maximize out-degree | 15 | Connectivity |
| 4 | Adaptive A* Escape | Width-weighted A* search | 50 | width_weight=2.0 |
| 5 | Multi-hop Greedy | 3-step lookahead scoring | 20 | lookahead=3 |

#### 3.10.3 GSES Loop Algorithm

The GSES loop operates as follows:

1. **Generate**: For each heuristic template $h_k$, generate an escape path $\mathbf{p}_k$ from current node $n_{curr}$ avoiding blocked edges $\mathcal{B}$.
2. **Score**: Evaluate each path by actual traversal cost: $C(\mathbf{p}_k) = \sum_{(u,v) \in \mathbf{p}_k} w(u, v)$, with blocked edges penalized by $10^4$.
3. **Execute**: Select the minimum-cost path $\mathbf{p}^* = \arg\min_k C(\mathbf{p}_k)$ and record its performance in the history buffer.
4. **Self-reflect**: Update template weights based on a rolling window of the last 5 performance records:

$$\alpha_k = \frac{1}{\max\left(\bar{C}_k^{(\text{last }5)}, 10^{-3}\right)}, \quad w_k = \frac{\alpha_k}{\sum_j \alpha_j}$$

where $\bar{C}_k^{(\text{last }5)}$ is the mean cost of heuristic $k$ over the last 5 invocations.

#### 3.10.4 LHHEngine Integration

The `LHHEngine` wraps the GSES loop with state verbalization and escape counting. Each invocation increments the global escape counter and logs the selected heuristic with its reflection weights.

### 3.11 ERFM Training via REINFORCE

The ERFM is trained using the REINFORCE policy gradient algorithm with an actor-critic architecture.

#### 3.11.1 Tour Cost Decomposition

The cost function explicitly separates the fixed component from the optimizable component:

$$C(\pi) = C_{service} + C_{transition}(\pi)$$

where $C_{service} = \sum_{i=1}^{N} w(e_i)$ is constant across all permutations, and $C_{transition}(\pi) = \sum_{k=1}^{K-1} T_{\pi_k, \pi_{k+1}} + T_{\pi_K, \pi_1}$ depends on the visit order.

#### 3.11.2 Rollout Sampling

During training, the encoder first computes edge embeddings $\mathbf{H} = \text{ERFM.encode}(\mathbf{X}, \mathbf{M}, \mathbf{g})$. The decoder then autoregressively constructs a tour $\pi$ by sampling from the softmax distribution at each step:

$$e_{\pi_t} \sim \text{Categorical}\left(\pi_{\theta}(\cdot \mid e_{\pi_{t-1}}, \mathbf{e}_{visited}, \mathbf{H}, \tau)\right)$$

The temperature $\tau$ anneals from 1.5 to 0.2 over the training episodes, transitioning from exploration to exploitation.

#### 3.11.3 Policy Gradient Update

The reward is the negative normalized cost: $R = -C(\pi) / (C_{service} \times 0.1)$. The advantage function is:

$$A_t = G_t - V(s_t)$$

where $G_t = \sum_{k=t}^{K} \gamma^{k-t} R$ is the discounted return with $\gamma = 0.99$. The training loss combines three components:

$$\mathcal{L} = \underbrace{-\mathbb{E}[\log \pi_{\theta}(a_t \mid s_t) \cdot \hat{A}_t]}_{\text{Policy Loss}} + 0.5 \cdot \underbrace{\mathbb{E}\left[(G_t - V(s_t))^2\right]}_{\text{Value Loss}} + 0.02 \cdot \underbrace{\mathbb{E}[\mathcal{H}(\pi_{\theta})]}_{\text{Entropy Bonus}}$$

where $\hat{A}_t$ is the standardized advantage, gradients are clipped to norm 1.0, and the model with the lowest training cost is checkpointed as the best model.

### 3.12 Integrated LE-DEGN Dynamic Execution System

The `LEDEGNSystem` orchestrates the three modules through a three-phase dynamic execution pipeline.

#### 3.12.1 Phase 1: Global Macro Planning

The ERFM generates an initial global route using 5 sampled rollouts with temperature $\tau = 0.5$, selecting the one with minimum transition cost. Each rollout uses a stochastic sampling-argmax hybrid: 70% categorical sampling for diversity, 30% greedy selection for quality. Blocked edges are excluded from the candidate set.

#### 3.12.2 Phase 2: Dynamic Execution with Re-planning

During traversal, three predefined congestion events inject random edge blockages (2-3 edges each, representing congestion waves). At each event:

1. **SA-DGWN Prediction**: The spatio-temporal model predicts congestion probabilities $\hat{\mathbf{p}} \in [0, 1]^N$. Edges with $\hat{p}_i > 0.5$ are flagged as high-risk.

2. **Path Interference Detection**: If the current tour intersects any blocked edge, the system enters ESCAPE state.

3. **LHH Escape**: The `LHHEngine` invokes the GSES loop to find an escape path from the blocked junction.

4. **ERFM Re-planning**: With higher temperature ($\tau = 0.8$) and blocked edges excluded, the ERFM generates a new global route.

5. **Road Recovery**: With 30% probability per event, one previously blocked edge is randomly reopened (simulating temporary congestion clearance).

#### 3.12.3 Phase 3: Local 2-opt Refinement

A bounded 2-opt local search refines the final tour by iteratively reversing subsequences:

$$\text{for } 1 < i < j < K: \quad \text{if } T_{\pi_{i-1},\pi_j} + T_{\pi_i,\pi_{j+1}} < T_{\pi_{i-1},\pi_i} + T_{\pi_j,\pi_{j+1}}: \quad \text{reverse } \pi[i:j+1]$$

The search uses incremental cost computation (only evaluating changed transitions) and terminates when no improving move is found or after 300 iterations.

#### 3.12.4 Final AOCC Computation

After optimization, the lower bound $L$ is computed from the nearest-neighbor heuristic on the transition cost matrix, and the upper bound $U$ uses a conservative estimate. The AOCC is calculated over the full execution trajectory.

### 3.13 Baseline Comparison Method: Greedy Nearest-Neighbor with DHAN-Style Congestion Handling

To establish a rigorous comparative baseline, we implement a **Greedy Nearest-Neighbor heuristic** augmented with **DHAN-style (Dynamic Heuristic Attention Network)** congestion handling. This baseline represents a strong conventional approach that, crucially, lacks the three defining innovations of LE-DEGN: learned edge representations, spatio-temporal congestion prediction, and adaptive heuristic selection.

#### 3.13.1 Nearest-Neighbor Route Construction

The baseline constructs routes directly from the transition cost matrix $\mathbf{T}$ using a multi-start greedy nearest-neighbor strategy:

1. Sample $m = \min(5, N)$ distinct starting edges.
2. For each start $s$, greedily append the nearest unvisited edge: $e_{next} = \arg\min_{e \notin \text{tour}} T_{curr, e}$.
3. Select the tour with minimum total transition cost across all starts.

$$\pi_{NN} = \arg\min_{s \in \mathcal{S}} C_{transition}(\text{greedy\_tour}(s, \mathbf{T}))$$

#### 3.13.2 DHAN-Style Congestion Penalty

The DHAN component models congestion impact through a fixed-percentage penalty mechanism. Upon encountering a congestion event that blocks any edge in the current tour:

$$\text{escape\_cost} = 
\begin{cases} 
0, & \text{if no tour edge is blocked} \\
0.15 \times C_{transition}(\pi), & \text{if any tour edge is blocked} 
\end{cases}$$

The total cost under congestion is:

$$C_{DHAN} = C_{service} + C_{transition}(\pi) + \sum_{e \in \text{events}} \text{escape\_cost}(e)$$

This approach represents the conventional paradigm of reactive congestion handling: it incurs a fixed penalty upon congestion detection but performs no predictive avoidance and no adaptive re-planning.

#### 3.13.3 Key Differences from LE-DEGN

| Aspect | LE-DEGN | Baseline (DHAN+NN) |
|:---|:---|:---|
| Route Construction | Learned via ERFM Transformer | Greedy nearest-neighbor |
| Congestion Response | Proactive prediction + adaptive escape (LHH) | Reactive fixed penalty (15%) |
| Re-planning | Full ERFM re-plan with blocked edge exclusion | No re-planning |
| Exploration | 5 sampled rollouts + stochastic decoding | 5 deterministic greedy starts |
| Optimization | 2-opt local refinement | None |
| Edge Representation | 13-D learned embeddings via self-attention | Raw transition cost matrix |

This baseline provides a fair and interpretable reference point: any performance advantage of LE-DEGN must be attributable to its learned representations, predictive capabilities, or adaptive mechanisms rather than to implementation-specific optimizations.

### 3.1 混合城市道路网络的数学公式化

城市道路系统建模为多向图：

$$\mathcal{G} = (\mathcal{V}, \mathcal{E}, \mathcal{W})$$

其中 $\mathcal{V} = \{v_1, v_2, \dots, v_n\}$ 为路口节点集合，$\mathcal{E} \subseteq \mathcal{V} \times \mathcal{V}$ 为多向边集合，$\mathcal{W}$ 为每条边 $e \in \mathcal{E}$ 上的空间、物理与语义属性向量集合。

节点坐标在微扰二维网格空间中分布。网格维度动态计算：

$$N_x = \max(3, \lceil\sqrt{|\mathcal{V}|}\rceil), \quad N_y = \max(3, \lceil |\mathcal{V}| / N_x \rceil)$$

节点 $v_k$ 坐标：$x_k = j \cdot s + \delta_x$，$y_k = -i \cdot s + \delta_y$，其中 $s = 50.0 / \max(N_x, N_y)$，$\delta_x, \delta_y \sim \mathcal{N}(0, (0.25s)^2)$。高斯微扰打破网格对称性，迫使求解器学习鲁棒的空间几何启发式特征。

### 3.2 拓扑路口采样与路网边生成

路口类型通过轮盘赌采样：$Type(v_k) \sim \text{Categorical}(0.07, 0.65, 0.18)$，对应断头路、T型路口、十字路口。最大度数限制：$\text{deg}_{max} \in \{1, 3, 4\}$。

沿中心列 $j = \lfloor N_x / 2 \rfloor$ 构建主干道（宽15-25m）。非主干道路段长度 $l_e \sim \mathcal{N}(80, 20^2)$ 米，走向角 $\theta_e = (\theta_{base} + \eta) \bmod 360$，其中 $\eta \sim \mathcal{N}(0, 15^2)$。边宽从 $\{2.5, 3.0, ..., 8.0\}$ 米的离散分布中采样。

### 3.3 语义道路场景分类与时间窗口约束

| 场景 | $\beta_e$ | $P_{e, cong}$ | 时间窗口 | 特征 |
|:---|:---:|:---:|:---|:---|
| 主干道 | 1.2 | 0.2 | [0, 1440] | 高速度，低拥堵 |
| 商业步行街 | 0.5 | 0.4 | [0, 1440] | 低速度，中等拥堵 |
| 学校周边 | 0.6 | 0.5 | [540, 1020] | 高峰拥堵 |
| 居民区 | 0.7 | 0.3 | [0, 1440] | 中等速度，低拥堵 |
| 施工路段 | 0.3 | 0.8 | [0, 1440] | 极低速度，高拥堵 |

空间相关采样分布：

$$P(\text{场景}) = \begin{cases} [0.1, 0.1, 0.5, 0.2, 0.1], & |j - c_j| \le 1 \\ [0.1, 0.1, 0.1, 0.6, 0.1], & \text{否则} \end{cases}$$

基本通行成本：$cost_{e, base} = l_e / \beta_e$。

### 3.4 物理运动学约束转化与转向惩罚

1. **单向化转换**：$w_e < 4.5$米 → 删除反向边。
2. **窄道阻尼**：$3.0 \le w_e < 4.5$米 → $cost_e = cost_{e, base} \times 1.5$。
3. **转向惩罚**：左转 $\Delta \tau_{left} = 1.5$秒，掉头 $\Delta \tau_{uturn} = 7.0$秒。

这些规则塑造了高度非线性的优化空间，极端掉头惩罚在数学上作为障碍参数，促使算法远离涉及过度掉头回溯的局部最优。

此处应有图表+[运动学约束与转向惩罚机制示意图]

### 3.5 强连通性修复机制

执行三阶段连通性修复：

**阶段1**：零度孤立节点 → 连接至最近活跃节点，添加双向补边（权重80s，宽度5m，全天候）。

**阶段2**：出度为0的汇节点 → 添加单向出边至最近邻居。

**阶段3**：强连通分量桥接 → 迭代连接最大SCC与次大SCC的最邻近节点对：

$$(u^*, v^*) = \arg\min_{u \in S_1, v \in S_2} \text{hypot}(x_u - x_v, y_u - y_v)$$

添加双向补边，重复至Tarjan算法验证全局强连通。修复后预计算Dijkstra全源最短路径矩阵 $\mathbf{D}$。

### 3.6 随时性能评估机制与AOCC指标

`PerformanceTracker` 记录收敛轨迹 $\mathcal{T} = \{(t_0, c_0), \dots, (t_m, c_m)\}$，在每次成本改进或固定间隔（0.05s）更新。

AOCC计算：时间预算 $T_{budget}$ 等分为 $N = 500$ 区间，在每个时间步 $t_k$：

$$c_{clamped}(t_k) = \max(L, \min(c(t_k), U))$$
$$y_{norm}(t_k) = \frac{c_{clamped}(t_k) - L}{U - L}$$
$$\text{AOCC} = \frac{1}{N} \sum_{k=1}^{N} (1.0 - y_{norm}(t_k))$$

下界 $L$ = 服务成本 + 每行最小转移成本之和（近邻下界）。上界 $U$ = 服务成本 + $300 \times$ 路径边数。AOCC接近1.0表示近瞬时收敛；接近0.0表示停滞。

### 3.7 线图构建与边中心表示

LE-DEGN的核心创新是将原始路图转换为线图（边对偶图），将路由问题重新表述为边上的节点排序问题。

#### 3.7.1 线图拓扑

原图 $\mathcal{G}$ 中每条有向边 $e = (u, v, k)$ 成为线图 $\mathcal{L}$ 的一个节点：

$$\mathcal{V}_L = \{e \in \mathcal{E}_{\mathcal{G}}\}$$

线图中从 $e_i$ 到 $e_j$ 的边存在于 $e_i$ 的终点等于 $e_j$ 的起点时：

$$\mathcal{E}_L = \{(e_i, e_j) \mid v_i^{dest} = v_j^{src}\}$$

此处应有图表+[原图到线图的转换示意图]

#### 3.7.2 边特征编码

每个线图节点 $e_i$ 编码为13维特征向量：

$$\mathbf{x}_i = [\frac{w_i}{200}, \frac{wid_i}{25}, \frac{\theta_i}{360}, type_i, \frac{x_u}{60}, \frac{y_u}{60}, \frac{x_v}{60}, \frac{y_v}{60}, \frac{d_{in}(u)}{10}, \frac{d_{out}(v)}{10}, \frac{tp_i}{7}, \frac{E_i}{1440}, \frac{L_i}{1440}]$$

#### 3.7.3 转移代价矩阵

核心优化目标编码于转移代价矩阵 $\mathbf{T} \in \mathbb{R}^{N \times N}$：

$$T_{ij} = \begin{cases} 0, & i = j \\ \mathbf{D}[v_i^{dest}, v_j^{src}] + \text{pen}_{turn}, & v_i^{dest} = v_j^{src} \\ \mathbf{D}[v_i^{dest}, v_j^{src}], & \text{否则} \end{cases}$$

其中 $\text{pen}_{turn} = (\text{turn\_penalty} - 1.0) \times 15.0$ 放大运动学惩罚。

#### 3.7.4 转弯惩罚矩阵

惩罚偏置矩阵 $\mathbf{M} \in \mathbb{R}^{N \times N}$ 引导注意力机制避开运动学上不利的转移：

$$M_{ij} = \begin{cases} -8.0, & \text{掉头（惩罚} \ge 7.0\text{）} \\ -1.5, & \text{左转（惩罚} \ge 1.5\text{）} \\ 0, & \text{否则} \end{cases}$$

### 3.8 边中心路由基础模型（ERFM）

ERFM作为全局路径规划主干，结合惩罚感知自注意力与边中心解码机制。路径 $\pi = (e_{\pi_1}, \dots, e_{\pi_K})$ 的总成本分解为：

$$C_{total}(\pi) = \underbrace{\sum_{e \in \pi} w(e)}_{\text{服务成本（固定）}} + \underbrace{\sum_{k=1}^{K-1} T_{\pi_k, \pi_{k+1}} + T_{\pi_K, \pi_1}}_{\text{转移成本（可优化）}}$$

由于服务成本对排列不变，优化简化为最小化转移成本。

#### 3.8.1 惩罚偏置多头注意力

注意力机制将惩罚矩阵作为加性偏置：

$$\text{Attention}(\mathbf{Q}, \mathbf{K}, \mathbf{V}, \mathbf{M}) = \text{softmax}(\frac{\mathbf{Q}\mathbf{K}^T}{\sqrt{d_k}} + \mathbf{M})\mathbf{V}$$

#### 3.8.2 Edge Transformer Block

$$\mathbf{h}' = \text{LayerNorm}(\mathbf{x} + \text{Dropout}(\text{Attn}(\mathbf{x}, \mathbf{M})))$$
$$\mathbf{h}_{out} = \text{LayerNorm}(\mathbf{h}' + \text{Dropout}(\text{FFN}(\mathbf{h}')))$$

FFN为双层GELU激活前馈网络。编码器含3个堆叠的Edge Transformer Block（$d_{model} = 64$，$n_{heads} = 4$，$d_{ff} = 128$）。

#### 3.8.3 全局属性嵌入

全局属性向量 $\mathbf{g} = [\bar{w}/25, 1.5, 7.0, 0.5] \in \mathbb{R}^4$ 通过双层MLP投影后广播-添加到所有边嵌入。

#### 3.8.4 解码器与价值估计

解码器自回归构建路径。给定当前边嵌入 $\mathbf{e}_{curr}$、上下文嵌入 $\mathbf{e}_{ctx}$（已访问边嵌入均值）和候选边嵌入 $\mathbf{e}_{cand}$：

$$q = \mathbf{W}_q \cdot [\mathbf{e}_{curr} \oplus \mathbf{e}_{ctx}]$$
$$\pi_{\theta}(e_{next} \mid \cdot) = \text{softmax}(q \cdot (\mathbf{W}_k \mathbf{e}_{cand})^T / (\sqrt{d_{model}} \cdot \tau))$$

并行价值头：$V(\mathbf{e}_{curr}, \mathbf{e}_{ctx}) = \text{MLP}([\mathbf{e}_{curr} \oplus \mathbf{e}_{ctx}])$。

### 3.9 时空注意力动态图波网络（SA-DGWN）

SA-DGWN预测每条路段的未来拥堵概率，实现主动路径适应。

#### 3.9.1 扩散因果卷积

$$h_t^{(l)} = \sum_{k=0}^{K-1} \mathbf{W}_k^{(l)} \cdot \mathbf{x}_{t - d^l \cdot k} + \mathbf{b}^{(l)}$$

其中 $d^l = 2^l$ 为膨胀率，在不增加参数的情况下扩展感受野。

#### 3.9.2 图卷积层

$$\mathbf{h}^{(spatial)} = \text{ReLU}(\mathbf{W} \cdot \hat{\mathbf{A}} \cdot \mathbf{x})$$

$\hat{\mathbf{A}} = \mathbf{A} / \mathbf{D}$ 为行归一化邻接矩阵。

#### 3.9.3 带门控的时空块

$$\mathbf{h}_{tcn} = \text{DilatedCausalConv}(\mathbf{x})$$
$$\mathbf{h}_{gcn} = \text{GraphConv}(\mathbf{h}_{tcn}[:, -1, :], \mathbf{A})$$
$$\mathbf{g} = \sigma(\mathbf{W}_g \cdot \mathbf{h}_{gcn})$$
$$\mathbf{h}_{out} = \text{LayerNorm}(\mathbf{h}_{gcn} \odot \mathbf{g})$$

Sigmoid门控 $\mathbf{g} \in [0, 1]^d$ 自适应调制每个特征维度。

#### 3.9.4 训练过程

在12个历史时间步的速度与流量特征上训练。标签从速度阈值推导（speed < 0.3 → 拥堵）。损失函数为二元交叉熵：

$$\mathcal{L}_{BCE} = -\frac{1}{N}\sum_{i=1}^{N} [y_i \log \hat{y}_i + (1 - y_i) \log(1 - \hat{y}_i)]$$

输出层使用Sigmoid激活产生拥堵概率 $\hat{p}_i \in [0, 1]$，$\hat{p}_i > 0.5$ 的边被标记为高风险。

### 3.10 语言驱动启发式中心（LHH）

LHH模块在动态拥堵阻塞当前路径时提供自适应逃逸规划，形式化为五个启发式模板上的"生成-评估-执行-反思"（GSES）循环。

#### 3.10.1 状态语言化

逃逸前，`StateVerbalizer` 构建局部子图（k-hop邻域）的结构化文本表示，包含节点位置、边宽度、权重和阻塞状态。

#### 3.10.2 启发式模板库

| # | 启发式 | 策略 | 最大深度 | 关键参数 |
|:---:|:---|:---|:---:|:---|
| 1 | 贪婪最宽道路 | 最大化边宽度 | 15 | 宽度优先 |
| 2 | 最小惩罚逃逸 | 最小化转弯惩罚×权重 | 15 | 左转×1.5, 掉头×7.0 |
| 3 | 最大连通性 | 最大化出度 | 15 | 连通性 |
| 4 | 自适应A*逃逸 | 宽度加权A*搜索 | 50 | width_weight=2.0 |
| 5 | 多跳贪婪 | 3步前瞻评分 | 20 | lookahead=3 |

#### 3.10.3 GSES循环算法

1. **生成**：对每个启发式模板 $h_k$，从当前节点生成避开阻塞边 $\mathcal{B}$ 的逃逸路径 $\mathbf{p}_k$。
2. **评估**：按实际遍历成本评分：$C(\mathbf{p}_k) = \sum_{(u,v) \in \mathbf{p}_k} w(u, v)$，阻塞边罚 $10^4$。
3. **执行**：选择最小成本路径 $\mathbf{p}^* = \arg\min_k C(\mathbf{p}_k)$ 并记录性能。
4. **反思**：基于最近5次性能记录的滑动窗口更新模板权重：

$$\alpha_k = \frac{1}{\max(\bar{C}_k^{(最近5次)}, 10^{-3})}, \quad w_k = \frac{\alpha_k}{\sum_j \alpha_j}$$

#### 3.10.4 LHHEngine集成

`LHHEngine` 将GSES循环包装在状态语言化和逃逸计数中，每次调用递增全局逃逸计数器并记录所选启发式及其反思权重。

### 3.11 ERFM训练：REINFORCE策略梯度

ERFM使用REINFORCE策略梯度算法与Actor-Critic架构进行训练。

#### 3.11.1 路径成本分解

$$C(\pi) = C_{service} + C_{transition}(\pi)$$

$C_{service}$ 对所有排列恒定，$C_{transition}(\pi)$ 依赖于访问顺序。

#### 3.11.2 采样Rollout

编码器计算边嵌入 $\mathbf{H}$，解码器自回归采样：

$$e_{\pi_t} \sim \text{Categorical}(\pi_{\theta}(\cdot \mid e_{\pi_{t-1}}, \mathbf{e}_{visited}, \mathbf{H}, \tau))$$

温度 $\tau$ 从1.5退火至0.2，从探索过渡到利用。

#### 3.11.3 策略梯度更新

奖励为归一化负成本：$R = -C(\pi) / (C_{service} \times 0.1)$。优势函数：$A_t = G_t - V(s_t)$，$G_t = \sum_{k=t}^{K} \gamma^{k-t} R$，$\gamma = 0.99$。

训练损失：

$$\mathcal{L} = -\mathbb{E}[\log \pi_{\theta}(a_t | s_t) \cdot \hat{A}_t] + 0.5 \cdot \mathbb{E}[(G_t - V(s_t))^2] + 0.02 \cdot \mathbb{E}[\mathcal{H}(\pi_{\theta})]$$

梯度裁剪至范数1.0。训练成本最低的模型被保存为最佳模型。

### 3.12 LE-DEGN集成动态执行系统

`LEDEGNSystem` 通过三阶段动态执行管线协调三个模块。

#### 3.12.1 阶段1：全局宏观规划

ERFM使用5次采样rollout（$\tau = 0.5$）生成初始全局路径，选择转移成本最低者。每次rollout采用随机采样-贪心混合策略（70%分类采样，30%贪心选择）。

#### 3.12.2 阶段2：动态执行与重规划

三波预定拥堵事件注入随机边阻塞（每次2-3条）。在每次事件中：

1. **SA-DGWN预测**：预测拥堵概率 $\hat{\mathbf{p}}$，$\hat{p}_i > 0.5$ 标记为高风险。
2. **路径干扰检测**：若当前路径与任何阻塞边相交 → 进入ESCAPE状态。
3. **LHH逃逸**：调用GSES循环寻找逃逸路径。
4. **ERFM重规划**：以更高温度（$\tau = 0.8$）并排除阻塞边，生成新全局路径。
5. **道路恢复**：每次事件以30%概率随机恢复一条先前阻塞的边。

#### 3.12.3 阶段3：局部2-opt优化

有界2-opt局部搜索通过迭代反转子序列细化最终路径：

$$\text{if } T_{\pi_{i-1},\pi_j} + T_{\pi_i,\pi_{j+1}} < T_{\pi_{i-1},\pi_i} + T_{\pi_j,\pi_{j+1}}: \text{ reverse } \pi[i:j+1]$$

搜索使用增量成本计算，无改进时或300次迭代后终止。

#### 3.12.4 最终AOCC计算

下界 $L$ 从转移代价矩阵的近邻启发式下界计算，上界 $U$ 使用保守估计。AOCC在整个执行轨迹上计算。

### 3.13 基线对比方法：贪心最近邻 + DHAN风格拥堵处理

为建立严格的对比基线，我们实现了**贪心最近邻启发式**，并增强以**DHAN风格（动态启发式注意力网络）拥堵处理**。该基线代表了一种强大的传统方法，但关键地缺少LE-DEGN的三个定义性创新：学习边表示、时空拥堵预测和自适应启发式选择。

#### 3.13.1 最近邻路径构建

基线直接从转移代价矩阵 $\mathbf{T}$ 使用多起点贪心最近邻策略构建路径：

1. 采样 $m = \min(5, N)$ 个不同起始边。
2. 对每个起点 $s$，贪心地追加最近未访问边：$e_{next} = \arg\min_{e \notin \text{tour}} T_{curr, e}$。
3. 在所有起点中选择最小总转移成本的路径。

$$\pi_{NN} = \arg\min_{s \in \mathcal{S}} C_{transition}(\text{greedy\_tour}(s, \mathbf{T}))$$

#### 3.13.2 DHAN风格拥堵惩罚

DHAN组件通过固定百分比惩罚机制建模拥堵影响。当遭遇阻塞当前路径任一边的拥堵事件时：

$$\text{escape\_cost} = \begin{cases} 0, & \text{无路径边被阻塞} \\ 0.15 \times C_{transition}(\pi), & \text{任一路径边被阻塞} \end{cases}$$

拥堵下的总成本：

$$C_{DHAN} = C_{service} + C_{transition}(\pi) + \sum_{e \in \text{events}} \text{escape\_cost}(e)$$

该方法代表了被动拥堵处理的传统范式：检测到拥堵时产生固定惩罚，但不执行任何预测性避让，也不进行自适应重规划。

#### 3.13.3 与LE-DEGN的关键差异

| 方面 | LE-DEGN | 基线（DHAN+NN） |
|:---|:---|:---|
| 路径构建 | 通过ERFM Transformer学习 | 贪心最近邻 |
| 拥堵响应 | 主动预测 + 自适应逃逸（LHH） | 被动固定惩罚（15%） |
| 重规划 | 完整ERFM重规划，排除阻塞边 | 无重规划 |
| 探索策略 | 5次采样rollout + 随机解码 | 5次确定性贪心起点 |
| 局部优化 | 2-opt局部细化 | 无 |
| 边表示 | 13维学习嵌入，通过自注意力 | 原始转移代价矩阵 |

该基线提供了一个公平且可解释的参考点：LE-DEGN的任何性能优势都必须归因于其学习表示、预测能力或自适应机制，而非实现特定的优化。
