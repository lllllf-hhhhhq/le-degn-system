## 第二章：问题建模

### 2.1 中国邮路问题（Route Inspection Problem）

**中国邮路问题（CPP）** 由管梅谷教授于1962年提出，核心问题是：一个邮递员从邮局出发，要走遍他负责的每条街道至少一次，最后返回邮局，怎样走才能使总路程最短？

**数学表述**：给定一个图 $G = (V, E)$，每条边 $e \in E$ 有一个权重 $w(e)$（通行代价），求一条闭合路径 $P$，使得：

- 每条边 $e \in E$ 至少被经过一次
- 总代价 $\sum_{e \in P} w(e)$ 最小

### 2.2 混合路网扩展（MCPP）

真实城市路网远比标准 CPP 复杂。LE-DEGN 解决的是**混合中国邮路问题（MCPP）**，增加了三个关键扩展：

1. **混合方向**：道路有单向（→）和双向（↔）之分，不能随意逆向行驶
2. **动态通行时间**：道路的通行时间不是固定的，会随拥堵情况实时变化
3. **异构道路类型**：不同道路有不同的速度因子和拥堵概率

### 2.3 优化目标公式

LE-DEGN 的优化目标是在满足约束的前提下，最小化总出行成本：

$$\min \sum_{e \in P} t(e)$$

**约束条件**：

- **覆盖约束**：每条边 $e \in E$ 至少被遍历一次
- **方向约束**：对于单向边 $(u \rightarrow v)$，只能从 $u$ 到 $v$ 方向行驶
- **连通性约束**：路径必须是连续的，不能出现"跳跃"
- **时间窗约束**：部分道路有时间限制（如学校周边仅在 9:00-17:00 有特殊拥堵）

**成本分解**：总成本 = 服务成本 + 转移成本

- **服务成本**（固定）：每遍历一条边所必须付出的代价，与访问顺序无关
- **转移成本**（可优化）：从一条边切换到另一条边之间的"绕路"代价，与访问顺序密切相关

优化重点在于**最小化转移成本**，因为服务成本是不可压缩的固定开销。

## 第三章：路网环境构建

### 3.1 网格生成 → 高斯扰动

路网生成采用**受控随机**策略，模拟历史城区的不规则路网特征：

1. **网格基底**：在 $S_x \times S_y$ 的二维网格上放置节点，其中 $S_x = \lceil\sqrt{N}\rceil$，$S_y = \lceil\frac{N}{S_x}\rceil$
2. **高斯扰动**：每个节点坐标加上随机偏移：
   $$x_i = j \times \text{step} + \mathcal{N}(0, \text{step} \times 0.25)$$
   $$y_i = -i \times \text{step} + \mathcal{N}(0, \text{step} \times 0.25)$$

   扰动打破规则网格的对称性，迫使模型学习鲁棒的空间几何特征，而不是死记网格规律。

3. **路口类型采样**：通过轮盘赌随机决定每个节点是断头路、T型路口还是十字路口

### 3.2 道路类型参数表

每条道路被赋予一种语义场景类型，影响其通行特性：

| 序号 | 道路类型 | 速度因子 | 拥堵概率 | 时间窗口 | 特征描述 |
|:----:|----------|:--------:|:--------:|:--------:|----------|
| 1 | **主干道** | 1.2 | 0.2 | 全天 | 宽阔高速，车道多，不易拥堵 |
| 2 | **商业步行街** | 0.5 | 0.4 | 全天 | 限速通行，人流量大，易拥堵 |
| 3 | **学校周边** | 0.6 | 0.5 | 09:00-17:00 | 上下学高峰时段严重拥堵 |
| 4 | **居民区** | 0.7 | 0.3 | 全天 | 中等速度，相对畅通 |
| 5 | **施工路段** | 0.3 | 0.8 | 全天 | 极低速度，频繁拥堵 |

- **速度因子**：大于 1 表示比标准速度快（通行时间更短），小于 1 表示更慢
- **拥堵概率**：该道路在模拟中发生拥堵的基础概率

### 3.3 通行时间计算公式

道路 $e$ 的实际通行时间由三个因素共同决定：

$$t(e) = w(e) \times \tau(e) \times \gamma(e)$$

其中：

| 符号 | 含义 | 说明 |
|:----:|------|------|
| $w(e)$ | 基础权重 | 道路长度相关的基础代价，$w(e) = \frac{\text{base\_length}}{\text{speed\_factor}}$ |
| $\tau(e)$ | 速度因子倒数 | $\tau(e) = \frac{1}{\text{speed\_factor}}$，速度越慢，$\tau$ 越大，通行时间越长 |
| $\gamma(e)$ | 拥堵因子 | $\gamma(e) \in \{1, 2\}$，正常通行时为 1，发生拥堵时翻倍为 2 |

**举例**：一条施工路段（speed_factor = 0.3）的基础长度 80m，理论通行时间 = $80 / 0.3 = 267$ 单位。如果发生拥堵（$\gamma = 2$），实际通行时间 = $267 \times 2 = 534$ 单位。

### 3.4 运动学约束

为模拟真实街景车的物理限制，引入以下约束：

1. **单向化转换**：宽度 < 4.5m 的窄路自动转为单向（删除反向边）
2. **窄道阻尼**：宽度在 3.0m ~ 4.5m 的道路，通行代价乘以 1.5
3. **转向惩罚**：
   - 左转：额外增加 1.5 倍惩罚
   - 掉头：额外增加 7.0 倍惩罚

这些惩罚使优化空间高度非线性，掉头的高额惩罚在数学上充当"障碍参数"，促使算法远离涉及频繁掉头的低效路径。

### 3.5 连通性修复三阶段

为确保路网中任意两点可达，执行三阶段修复：

```
阶段 1: 孤立节点补全
  零度节点 → 连接至最近活跃节点，添加双向补边

阶段 2: 汇节点修复
  出度为0的节点 → 添加单向出边至最近邻居

阶段 3: 强连通增强
  迭代连接最大SCC与次大SCC的最近节点对
  → 重复直到 Tarjan 算法验证全局强连通
```

修复后预计算 Dijkstra 全源最短路径矩阵，为后续转移代价计算提供基础。

## 第四章：线图转换（核心变换）

### 4.1 为什么要转线图？

这是 LE-DEGN 最核心的思想转换：

| | 原始图（点图） | 线图（边对偶图） |
|---|---|---|
| **视角** | 以路口为中心 | 以道路段为中心 |
| **问题** | 中国邮路问题（边遍历） | 哈密顿路径问题（节点排序） |
| **建模对象** | 遍历每条边 | 给所有"线图节点"排序 |
| **适合的模型** | 传统图算法 | Transformer / 注意力机制 |

**核心洞察**：Transformer 架构天然擅长处理"序列排序"问题。线图转换将"选哪条边走"的图遍历问题，变成"这些边节点怎么排"的序列决策问题，让 Transformer 的注意力机制能直接发挥作用。

### 4.2 线图定义

给定原图 $G = (V, E)$，其线图 $L(G) = (V', E')$ 定义为：

- **节点集** $V'$：原图中的每条有向边 $e \in E$ 成为线图中的一个节点
  $$V' = \{v'_e \mid e \in E\}$$

- **边集** $E'$：如果原图中边 $e_1$ 的终点等于边 $e_2$ 的起点，则在线图中 $v'_{e_1}$ 到 $v'_{e_2}$ 有一条边
  $$E' = \{(e_1, e_2) \mid \text{tail}(e_2) = \text{head}(e_1)\}$$

**直观理解**：如果你刚走完"A→B"这条路，接下来可以走"B→C"这条路（因为A→B的终点B正好是B→C的起点），那么在线图中，节点"A→B"和节点"B→C"之间就有一条边。

### 4.3 转移代价

线图中从节点 $i$（对应原边 $e_i$）到节点 $j$（对应原边 $e_j$）的转移代价为：

$$TC(i \to j) = SP\_cost(v_i, u_j) + TP(i, j)$$

其中：

| 分量 | 含义 | 计算方式 |
|:----:|------|----------|
| $SP\_cost(v_i, u_j)$ | **绕行代价** | 从边 $i$ 的终点到边 $j$ 的起点的最短路径代价（Dijkstra） |
| $TP(i, j)$ | **转弯惩罚** | 当两边直接衔接时，根据转弯角度施加惩罚：直行=1.0，左转=1.5，掉头=7.0 |

当 $v_i = u_j$（即两边直接衔接）时，转移代价主要来自转弯惩罚；当两边不直接衔接时，需要先"绕路"到目标边的起点，再考虑转弯。

### 4.4 线图特征提取（13维特征向量）

每个线图节点（即原图的一条边）被编码为 **13 维特征向量**：

| 维度 | 特征名称 | 含义 | 归一化方式 |
|:----:|----------|------|------------|
| 1 | 边权重 | 通行代价 | $w(e) / 200$ |
| 2 | 道路宽度 | 物理宽度 | $width / 25$ |
| 3 | 走向角度 | 道路方向 | $angle / 360$ |
| 4 | 道路类型 | 0=标准, 1=特殊 | 原始值 |
| 5 | 起点 x 坐标 | 起点空间位置 | $pos_x / 60$ |
| 6 | 起点 y 坐标 | 起点空间位置 | $pos_y / 60$ |
| 7 | 终点 x 坐标 | 终点空间位置 | $pos_x / 60$ |
| 8 | 终点 y 坐标 | 终点空间位置 | $pos_y / 60$ |
| 9 | 起点入度 | 该路口有多少条路进入 | $in\_deg / 10$ |
| 10 | 终点出度 | 该路口有多少条路出去 | $out\_deg / 10$ |
| 11 | 转弯惩罚 | 掉头惩罚因子 | $penalty / 7$ |
| 12 | 时间窗起始 | 最早可通行时间 | $E / 1440$ |
| 13 | 时间窗结束 | 最晚可通行时间 | $L / 1440$ |

## 第五章：核心模型

### 5.1 ERFM — 边中心路由基础模型

ERFM（Edge-centric Routing Foundation Model）是系统的路径规划"大脑"，负责学习全局最优的边遍历顺序。

#### 架构概览

```
┌─────────────────────────────────────────────────────────┐
│                    ERFM 架构                              │
│                                                          │
│  输入: 13维边特征 X ∈ ℝ^(N×13)                           │
│         │                                                │
│         ▼                                                │
│  ┌──────────────────────────────────────┐               │
│  │           输入投影层                   │               │
│  │     Linear(13→64) + LayerNorm + ReLU  │               │
│  └──────────────┬───────────────────────┘               │
│                 │                                         │
│  ┌──────────────▼───────────────────────┐               │
│  │    Edge Transformer Block ×3          │               │
│  │  ┌─────────────────────────────────┐ │               │
│  │  │ 惩罚偏置多头自注意力 (4 heads)    │ │               │
│  │  │    + LayerNorm + Dropout         │ │               │
│  │  ├─────────────────────────────────┤ │               │
│  │  │ 前馈网络 FFN (64→128→64)         │ │               │
│  │  │    GELU + Dropout + LayerNorm    │ │               │
│  │  └─────────────────────────────────┘ │               │
│  └──────────────┬───────────────────────┘               │
│                 │                                         │
│                 ▼                                         │
│  输出: 边嵌入矩阵 H ∈ ℝ^(N×64)                            │
│         │                                                │
│         ▼                                                │
│  ┌──────────────────────────────────────┐               │
│  │          解码器 (自回归)               │               │
│  │   当前边嵌入 + 上下文嵌入 → 候选概率   │               │
│  │   价值头: 并行评估当前状态价值         │               │
│  └──────────────────────────────────────┘               │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

#### 核心公式

**惩罚偏置注意力**：将转弯惩罚矩阵 $M_{penalty}$ 作为加性偏置注入注意力计算，使模型在学习阶段就"知道"哪些边之间的转移代价高：

$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}} + M_{penalty}\right)V$$

其中 $M_{penalty}[i, j]$：
- 掉头（150°~210°）：$-8.0$
- 左转（60°~120°）：$-1.5$
- 其他：$0.0$

负值越大，注意力权重越低，模型越倾向于避开这类转移。

**解码器**：自回归地逐条选择下一个要遍历的边：

$$P(\text{next} = j) = \text{softmax}\!\left(\frac{W_q[\text{curr} \parallel \text{ctx}] \cdot W_k(\text{candidates})^T}{\sqrt{d_k} \cdot T}\right)$$

其中 $T$ 是温度参数（从 1.5 退火到 0.2），控制探索与利用的平衡。

**训练：REINFORCE 策略梯度**

- 奖励：$R = -\text{cost} / \text{normalization}$（负成本 → 奖励）
- 优势函数：$A = R_{\text{return}} - V_{\text{value}}$（减去基线降低方差）
- 损失：$\mathcal{L} = \mathcal{L}_{\text{policy}} + 0.5\mathcal{L}_{\text{value}} + 0.02\mathcal{L}_{\text{entropy}}$

```
│  │  │ 图卷积层 (GraphConv)            │  │               │
│  │  │   H = ReLU(Â · X · W)          │  │               │
│  │  │   Â = D⁻¹A (行归一化邻接矩阵)    │  │               │
│  │  └──────────────┬─────────────────┘  │               │
│  │                 ▼                     │               │
│  │  ┌────────────────────────────────┐  │               │
│  │  │ 门控机制 (Gating)               │  │               │
│  │  │   z = tanh(Θ₁ * x) ⊙ σ(Θ₂ * x)│  │               │
│  │  └────────────────────────────────┘  │               │
│  └──────────────────────────────────────┘               │
│         │                                                │
│         ▼                                                │
│  输出: 拥堵概率 p ∈ [0,1]^N                               │
│        p > 0.5 → 高风险路段                               │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

#### 核心公式

**门控 TCN（时间卷积网络）**：

$$z = \tanh(\Theta_1 * x) \odot \sigma(\Theta_2 * x)$$

其中 $\odot$ 表示逐元素乘法，$\tanh$ 分支提取特征，$\sigma$（Sigmoid）分支作为门控开关，自适应地控制哪些时间特征通过。

**图卷积**：

$$H = \text{ReLU}(\hat{A} \cdot X \cdot W)$$

其中 $\hat{A} = D^{-1}A$ 是行归一化的邻接矩阵，$D$ 是度矩阵。

**训练**：使用二元交叉熵损失，标签从速度阈值推导（speed < 0.3 → 拥堵）。

### 5.3 LHH — 语言超启发式引擎

LHH（Language-driven Hyper-Heuristic Hub）是系统的"应急指挥官"，在路径被突发拥堵阻断时动态选择最优逃逸策略。

#### 五类启发式模板

| # | 名称 | 策略 | 最大深度 | 适用场景 |
|:-:|------|------|:--------:|----------|
| 1 | **贪婪最宽道路** | 优先选择宽度最大的边 | 15 | 狭窄的老城区，需要宽阔道路保证通行 |
| 2 | **最小惩罚逃逸** | 最小化转弯惩罚×权重 | 15 | 严格转弯限制的单行道系统 |
| 3 | **最大连通性** | 优先选择出度最高的节点 | 15 | 避免陷入死胡同，保持路径灵活性 |
| 4 | **自适应 A\* 逃逸** | 宽度加权 A\* 搜索 | 50 | 需要全局最优的长距离绕行 |
| 5 | **多跳贪婪** | 3步前瞻评分 | 20 | 局部绕行，计算开销有限 |

#### GSES 循环（生成-评分-执行-反思）

```
┌─────────────────────────────────────────────────────────┐
│                 GSES 循环                                 │
│                                                          │
│  ┌──────────┐    ┌──────────┐    ┌──────────┐           │
│  │ 1.生成   │───▶│ 2.评分   │───▶│ 3.执行   │           │
│  │ 对每个   │    │ 计算实际 │    │ 选择最小 │           │
│  │ 启发式   │    │ 遍历代价 │    │ 代价路径 │           │
│  │ 生成路径 │    │ 排序评估 │    │ 执行逃逸 │           │
│  └──────────┘    └──────────┘    └────┬─────┘           │
│                                       │                  │
│                          ┌────────────▼──────────┐      │
│                          │ 4.反思                 │      │
│                          │ 根据最近5次性能记录     │      │
│                          │ 更新各启发式权重        │      │
│                          │                        │      │
│                          │ w_h = 1/mean(cost[-5:])│      │
│                          │ 归一化后作为下次选择偏好 │      │
│                          └────────────────────────┘      │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

**反思权重更新公式**：

$$w_h = \frac{1}{\text{mean}(cost_h[-5:])}$$

归一化后，表现好的启发式获得更高权重，在后续决策中更可能被选中。

## 第六章：AOCC 评估指标

### 6.1 收敛曲线

AOCC（Area Over Convergence Curve，收敛曲线上方面积）衡量的是**算法在有限时间预算内的优化效率**。

```
成本
  │
U ┤╲
  │ ╲
  │  ╲___
  │      ╲___         ← 收敛曲线（随时间的成本下降轨迹）
  │         ╲___
  │            ╲___
L ┤               ╲___
  │
  └──────────────────────▶ 时间
  0                    T
```

### 6.2 AOCC 计算公式

在时间预算 $T$ 内等分为 $500$ 个区间，对每个时间步 $t_k$：

$$AOCC = \frac{1}{500} \sum_{k=1}^{500} \left(1 - \frac{cost(t_k) - L}{U - L}\right)$$

其中：

| 符号 | 含义 | 计算方式 |
|:----:|------|----------|
| $cost(t_k)$ | 时刻 $t_k$ 的最佳成本 | 从性能追踪器中读取 |
| $L$ | 理论下界 | 服务成本 + 每行最近邻转移代价之和 |
| $U$ | 理论上界 | 服务成本 + 路径长度 × 300 |

**直观理解**：AOCC 就是曲线上方那一块"空白区域"的面积占比。收敛越快、最终成本越低，AOCC 就越大。

### 6.3 cost 和 AOCC 的关系

| 维度 | cost（总成本） | AOCC（收敛曲线面积） |
|:----:|:-------------:|:-------------------:|
| **方向** | **越小越好 ↓** | **越大越好 ↑** |
| **衡量什么** | 最终解的质量（绝对值） | 优化的速度和效率（相对值） |
| **受什么影响** | 路径本身的好坏 | 初始成本、收敛速度、最终成本 |
| **类比** | 考试得了多少分 | 提分速度有多快 |

**两者是同一个东西的两个侧面**：

- **cost 低** = 最终找到了好路线
- **AOCC 高** = 找到好路线的过程又快又稳

一条理想的收敛曲线会快速下降并稳定在接近下界 $L$ 的位置，此时 cost 低、AOCC 高。如果算法收敛很慢或陷入局部最优，cost 高、AOCC 低。

**AOCC 取值范围**：

- $AOCC \approx 1.0$：近乎瞬时收敛到最优解（理想情况）
- $AOCC \approx 0.5$：在时间预算内实现了较好的优化
- $AOCC \approx 0.0$：几乎没有优化，收敛停滞

## 第八章：关键要点总结

### 8.1 三个核心创新点

| 创新点 | 解决的问题 | 核心方法 |
|:------:|------------|----------|
| **线图转换 + ERFM** | 传统模型无法处理"边遍历"问题 | 将边转换为节点，用 Transformer 自注意力学习边间转移关系 |
| **SA-DGWN 主动预测** | 传统方法在拥堵发生后被动响应 | 利用时空图卷积提前预测拥堵传播，在拥堵发生前就调整路线 |
| **LHH 自适应逃逸** | 单一启发式无法应对多变路况 | LLM 引擎在5种逃逸策略中动态选择，并通过反思机制持续优化偏好 |

### 8.2 实验结论

1. **成本降低显著**：LE-DEGN 相比基线方法（贪心最近邻 + DHAN 拥堵处理）平均降低 12.1% 总成本，在 80 节点网络达到峰值 16.3%
2. **三模块协同**：消融实验证实，单独 ERFM 无法应对动态环境（AOCC = 0.0），加入 LHH 降低 52.2% 成本，再加入 SA-DGWN 进一步降低 18.3%
3. **可扩展性好**：执行时间随网络规模近乎线性增长（0.82s ~ 2.49s），具备实际部署可行性
4. **拥堵鲁棒**：在不同拥堵级别（10%~30%）下保持稳定性能，预测优势随拥堵加剧而增大

### 8.3 常见问题解答

**Q1：LLM 在这里具体是什么？**

A：LHH 模块中的"语言"部分体现为 **StateVerbalizer（状态语言化）**——将当前局部路网结构（节点位置、边宽度、通行状态）转化为结构化的文本描述，然后由 GSES 循环基于这些文本信息在5种启发式模板中选择最优逃逸策略。系统的反思机制也类似于 LLM 的"从经验中学习"。

**Q2：AOCC 到底怎么看？cost 和 AOCC 矛盾吗？**

A：不矛盾。cost 是"结果好不好"，AOCC 是"过程快不快"。最优情况是 cost 低、AOCC 高。如果 cost 低但 AOCC 也低，说明虽然最终找到了好解，但收敛很慢；如果 AOCC 高但 cost 高，说明路径本身不够好。

**Q3：cost 越小越好，AOCC 越大越好，为什么？**

A：cost 是你要花的钱/时间，当然越少越好。AOCC 衡量的是"你多快找到便宜方案"——收敛曲线越陡峭、越早接近下界，AOCC 越大。所以 cost↓ 和 AOCC↑ 是好算法的两个并行特征。

**Q4：为什么一定要转线图？直接用原始图不行吗？**

A：原始图的节点是"路口"，边是"道路段"。中国邮路问题要求遍历所有边，而 Transformer 擅长处理"序列排序"问题。线图转换后，每条边变成节点，路径规划变成"给这些节点排序"，Transformer 的注意力机制就能直接学习"哪些边之间切换代价小"。如果直接用原始图，Transformer 需要间接推理边的遍历顺序，效率会大幅下降。

**Q5：LLM 和 GPU 有什么关系？**

A：LE-DEGN 的 LHH 模块目前运行在 CPU 上，使用结构化状态语言化而非真正的 LLM 推理。ERFM 和 SA-DGWN 的神经网络计算默认也在 CPU 上运行（以保证可复现性和可及性）。如果部署到 GPU 上，Transformer 的矩阵运算会显著加速，但当前设计中深度 LLM 集成（如用 GPT 生成新的启发式）是未来方向，需要 GPU 支持。
